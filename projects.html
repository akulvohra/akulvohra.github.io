Projects I've worked on either for hackatons or on the side:

  • Natural Language Recommendations: This was a project created for the 2019-20 #TFWorld 2.0 Challenge. The idea for this project was to create a new scientific paper search algorithm that would be capable of finding more relevant results than those that are already on the market. We used various different libraries and tools to complete this project. We used 1.3 million papers from Semantic Scholar's Open Corpus API as data. We used word2Vec CBOW to create citation vectors and labels. HuggingFace's TF2.0 implementation of BERT was used to create our model, with the citation vectors as the labels. A team member even made a library that allowed us to do faster similarity searches on TPU's called TPU-Index. Our process, as well as the code behind it is explained in depth in our GitHub repository. My role, in particular, was designing the process for filtering and pruning the data. All in all, the project was successful and especially showed its capabilities when looking for papers in niche topics given a paper abstract. Demos and notebooks can be found in the GitHub or the bottom of the DevPost submission. [Devpost Submission] [GitHub] [Video Demo]

  • Mind+: This was a project created for OmniHacks 2019. The idea for this project was to create a platform in which those with degrading mental ability due to age or other brain injury could use various digital mind exercises to keep their mind "sharp". My team implemented a matching task to affirm short term memory as well reaction time for hand eye coordination and dexterity. Backend built with: Python, Flask, SQLAlchemy. Frontend: HTML, CSS, JS. [Devpost Submission] [Project Website]

  • Glass Chemical Detection: A starter project made comparing the effectiveness of logistic regression and KNN classifier to classify glass based on chemical composition. Also experimented with confusion matrices to display results. [GitHub]

  • Sentwiment: The initial idea for this project was to use a sentiment analysis model on tweets to be able to analyze the community's current sentiment on a topic. Possible use cases of this could be to predict effectiveness and popularity during campaigning and election or to predict stock growth/decline. I used TextBlob, a simple language library that has inbuilt sentiment analysis. The Twitter API was accessed using TweePy. Project built with TextBlob, TweePy, Flask, JS, HTML/CSS. [GitHub]

  • Historical Tours: This project was created for TinoHacks 2018. The project's primary goal was to build a platform by which student's can learn about history through an interactive simulation. The simulation consists of tour stops on which the user would stop on and be given information via sound. The initial prototype was built using Unity3D. [Devpost Submission] [GitHub]
